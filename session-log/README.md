# Session Log

> Append-only журнал действий. Каждый день = один файл `YYYY-MM-DD.md`.
> **НЕ редактировать прошлые записи.** Только append.

---

## Зачем это нужно

Когда новый чат claude.ai или Claude Code сессия начинается, нужно понять:
- Что было сделано вчера/на прошлой неделе?
- Какие commits относятся к какой задаче?
- Какие решения были приняты и почему?

`docs/current-status.md` отвечает «где мы сейчас». 
**Этот лог отвечает «как мы сюда пришли»**.

Wiki handoff файлы — для residual state между терминалами одной задачи.
**Этот лог — для cross-session knowledge** для следующего чата.

---

## Когда писать

✅ После любого `git push origin main` в sdar-v2 ИЛИ sdar-v2-wiki
✅ После принятия архитектурного решения (даже без коммита)
✅ После завершения cluster phase
✅ После discovery (нашли проблему / нашли что что-то уже сделано)
✅ После запуска нового tool/integration

❌ НЕ писать для одиночного typo fix
❌ НЕ писать draft / WIP — только завершённое

---

## Формат записи

Файл `YYYY-MM-DD.md`. Если файла нет — создать. Если есть — append в конец.

Каждая запись:

```markdown
### HH:MM — <короткий заголовок>

**Что сделано:**
<1-3 строки прозы>

**Commits:**
- `abc1234` (sdar-v2): commit message
- `def5678` (wiki): commit message

**Затронуло:**
- Files changed: <pattern или конкретные файлы>
- Pages affected: <число или scope>

**Решения / discovery:**
- <что узнали или решили, если есть>

**Следующий шаг:**
<что планируется дальше, если применимо>

---
```

## Кто пишет

**Claude Code terminal** — после каждого commit + push:
1. Открывает `session-log/<today>.md` (создаёт если нет)
2. Append запись по формату выше
3. Сразу commit: `git commit -m "log: <дата> <короткое описание>"`

**Orchestrator chat (claude.ai)** — после принятия решения / discovery:
1. Просит Roman добавить запись через T1 prompt:
   ```
   Append to session-log/YYYY-MM-DD.md:
   ### HH:MM — заголовок
   <содержимое>
   ```

**Roman** — не пишет вручную, всё делегируется Claude Code.

---

## Когда читать

- 🟢 **Каждый новый чат claude.ai** — последние 3-5 записей текущей недели
- 🟢 **Resume after pause** — читать с момента последней работы по сегодня
- 🟢 **Looking up «когда мы это сделали»** — grep по этому каталогу
- 🟢 **Cluster post-mortem** — agg все записи касающиеся cluster

---

## Retention

- Сохраняем **навсегда** — git history бесплатен, contextual gold
- Старше 90 дней — можно перенести в `session-log/archive/YYYY-MM/` если корень засоряется
- Никогда не удалять — это память проекта

---

## Грубое правило

Если в новом чате claude.ai говоришь Roman «давай продолжим с того места, где остановились» — этот каталог должен дать ответ за 30 секунд чтения.

Если ответа нет — мы недостаточно писали в лог.

---

## Анти-паттерны

❌ **Простыни прозы.** Запись = 5-10 строк max.
❌ **«Поговорили о photo strategy».** Что РЕШИЛИ — пиши.
❌ **«Обновил 50 файлов».** Какой scope, какой commit.
❌ **Дублирование `current-status.md`.** Лог = действия, status = состояние.
❌ **Эмоции / благодарности.** Это git log, не diary.
